v0.102: Se porta desde MediaTools_v2 el control de carga de CPU en el footer de la ventana principal (label `CPU` + dropdown de presets High/Medium/Low/Minimal) y se cablea al backend Python con `--workers` para ajustar el paralelismo real del analisis; tambien se agrega el checkbox `Keep this window on top` con persistencia de estado. En la tabla de resultados se habilita resize manual de columnas desde headers (`Interactive`), se corrige el comportamiento de `Folder` para evitar truncado permanente (scroll horizontal + tooltip de path completo), y se incorpora `ShotNameDetector` para detectar nombre de shot y colorear/intercalar prefijos en `Sequence` con los dos colores usados en MediaTools. [SeqChecker - Control CPU y mejoras de tabla]

v0.101: Se agrega el listado copiable de frames corruptos. En la tabla de resultados, la celda `Corrupt` de cada secuencia con corruptos ahora es clickeable (subrayada) y abre un dialogo con los paths completos de esos frames, seleccionables y con boton `Copy all` para enviarlos a un tercero. El texto de status final muestra el total de corruptos como link (`N corrupt frames`) que abre el listado global de todos los frames corruptos detectados. El backend Python emite los paths en `corrupt_files`/`suspect_files` dentro de `MT_SEQ_RESULT`. [SeqChecker - Listado copiable de frames corruptos]

v0.1: Version inicial de LGA SeqChecker. App Qt C++ + backend Python para detectar frames EXR corruptos. Ventana unica con drag&drop de una zona (overlay reusado de MediaTools); al soltar carpeta(s) se escanean recursivamente todas las secuencias `.exr`, se agrupan por patron `nombre.####.exr` y se analizan en tres capas: estructural (gaps + outliers de tamano), header (`exrheader`) y decode completo (`exrcheck`) en pool de workers. Resultado en tabla por secuencia con conteo OK/Suspect/Corrupt y estado coloreado. [SeqChecker - Version inicial]

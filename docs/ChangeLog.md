v0.104: El control de CPU del footer ahora se aplica EN VIVO durante un analisis en curso, igual que MediaTools. Antes el numero de workers quedaba congelado al lanzar el proceso Python y solo cambiaba en el proximo drop. Ahora C++ escribe el limite elegido en un control-file (`cpu_control.txt` en AppData) que se le pasa al backend con `--control-file`; al mover el dropdown durante el analisis, C++ reescribe ese archivo. El backend (`LGA_SeqChecker.py`) crea el pool con un tope maximo (`--max-workers`) y limita la concurrencia real con un `DynamicGate` cuyo cupo relee un thread watcher cada 0.2s; al bajar el limite, los frames ya en proceso terminan y recien ahi se frena el resto, al subirlo arrancan mas de inmediato. Validado con test instrumentado: 8 concurrentes antes del cambio, 2 despues, sin esperar al fin del job. [SeqChecker - Control de CPU dinamico en vivo]

v0.1: Version inicial de LGA SeqChecker. App Qt C++ + backend Python para detectar frames EXR corruptos. Ventana unica con drag&drop de una zona (overlay reusado de MediaTools); al soltar carpeta(s) se escanean recursivamente todas las secuencias `.exr`, se agrupan por patron `nombre.####.exr` y se analizan en tres capas: estructural (gaps + outliers de tamano), header (`exrheader`) y decode completo (`exrcheck`) en pool de workers. Resultado en tabla por secuencia con conteo OK/Suspect/Corrupt y estado coloreado. [SeqChecker - Version inicial]
